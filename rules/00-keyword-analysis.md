# 00 キーワード分析ルール

`main_keyword` と `related_keywords` から、検索意図、顕在ニーズ、潜在ニーズ、読者の不安、最終判断、想定ペルソナ、記事タイプ、主目的、タイトル案3案、採用タイトル、メタディスクリプション、目標文字数、スラッグ案を自動生成する。

- 関連キーワードは本文へ詰め込まず、検索意図、見出し、比較表、FAQへ役割ごとに割り振る。
- 記事タイプは「おすすめ、比較、評判、口コミ、サービス解説、商品解説、車種解説、購入、売却、買取、料金、地域、制度、手続き、トラブル解決、ハウツー」などから判断する。
- 評判・口コミ系語（評判、口コミ、レビュー、悪評、しつこい、怪しい、おすすめしない、デメリット）がある場合は、架空口コミを作らず公式情報と個人感想を分ける。
- おすすめ・比較系語（おすすめ、ランキング、比較、人気、選び方）がある場合は、選定基準と根拠を明示し、根拠のない順位を付けない。
- 分析結果は `research.md` と `input.yml`、最終的に `metadata.json` へ反映する。
