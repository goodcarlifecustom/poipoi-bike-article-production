# 01 見出し・競合調査ルール

検索対象は `main_keyword`、重要度の高い `related_keywords`、検索意図を補うクエリから3〜5件に絞る。`npm run extract -- --slug {slug}` はこの方針で `serp.md`、`headings.csv`、`heading-analysis.md` を作成する。

`research.md` または調査ファイルへ、検索クエリ、調査日、競合タイトル、競合URL、主なH2/H3、共通論点、不足情報、記事で追加する判断材料、一次情報、確認できなかった事項を記録する。

検索失敗時は競合内容を推測しない。取得できなかったURL、理由、再調査に必要な `reference_urls` を記録する。
